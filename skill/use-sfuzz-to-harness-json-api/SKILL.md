`sfuzz` is a local CLI tool that allow black box fuzzing for API that are JSON only.

CRITICAL: Do not launch `sfuzz` without prior validation if the endpoint is allowed and ready to received some fuzzing.
CRITICAL: Do not use on production environment.

The overall description and usage of the local tool is always readily available via the command `sfuzz -h`.

Important: Ensure to always read the latest help and usage for the local tool.

## Install the tool

Prerequisite, have Go locally, then run:

```console
go install github.com/simcap/sfuzz/cmd/sfuzz
```

Then run `sfuzz -h` to ensure proper installation and read the latest description and usage. 

## Fuzz file

The tool usually needs a fuzz file with a strict syntax in order to launch a
fuzz run on a given JSON API.

The fuzz file is a simple text format with each line being requests describing endpoint to be fuzzed:

```text
[GET|PUT|POST|DELETE|PATCH] URL [{JSON Body}|@LOCAL_FILENAME]
```

Basically each line consists of:

1. a optional HTTP verb (default to GET)
2. a URL that must contains the scheme (http or https), a valid host to reach
3. a optional JSON body either inline or pointing to a local file (when starts with `@`)

The fuzz file allows also empty lines and comment lines if they start by `#`.

Example:

```text
GET https://nice.fr/FUZZUID?name=FUZZSTR&stamp=FUZZTME
POST https://nice.fr/FUZZUID?name=FUZZSTR&stamp=FUZZTME {"age": FUZZNUM, "date": "FUZZDTE" }
```

## Fuzz keywords

A fuzz keyword is a placeholder starting with `FUZZ` and terminating with a 3 charactes types (ex: `NUM`)
with an optional example value within it.

#### Types supported

| Fuzz Keyword | Type (i.e. format)   |
|--------------|----------------------|
| FUZZSTR      | Generic string       |
| FUZZNUM      | Numeral              |
| FUZZUID      | Universal identifier |
| FUZZDTE      | Date format          |
| FUZZTME      | Time or stamp format |
| FUZZJWT      | JWT auth token       |


#### Example value

A fuzz keyword can embed a example value within it which is considered a happy path value.

| Keyword with embedded values | Example value |
|------------------------------|---------------|
| FUZZjohnSTR                  | john          |
| FUZZ1234,34NUM               | 123,34        |
| FUZZav34ef-56cafUID          | av34ef-56caf  |
| FUZZ2024-03-05DTE            | 2024-03-05    |
| FUZZ12:43TME                 | 12:43         |

The embedded example serves various purposes for later fuzz run:

1. replace keyword with a happy path value when needed
2. use example value as a starting point to mutate and fuzz with exotic values

> [!TIP]
> Fuzz types and embedded examples are autogenerated when using Open API spec.

This syntax will make use of fuzz keywords (i.e. placeholder) in order to encode for later fuzzing:

1. The format/type/kind of the value to be fuzzed.
2. An example value that allow to have a happy path value during a run or to use the example to mutate and find relevant exotic values.

## Generate a fuzz file

One can generate a fuzz file by just writing it on disk if one know the JSON API endpoint specification.

Or we available one can read the Open API specification of the endpoint then generate the fill fuzz file.
